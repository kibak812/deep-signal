# 딥 시그널

`딥 시그널`은 가라앉은 데이터 해역을 탐사하며 신호 카드로 덱을 만드는 웹 기반 싱글플레이 덱 빌딩 로그라이크입니다. 모든 콘텐츠, 세계관, 카드, 유물, 적, 이벤트, 화면 구성은 독창적으로 제작되었습니다.

## 실행

```bash
npm run dev
```

브라우저에서 `http://127.0.0.1:4173` 또는 `http://localhost:4173`을 엽니다.

## 검증

```bash
npm test
npm run build
npm run audio:mix
npm run copy:audit
npm run playtest
npm run balance
npm run balance:long
npm run audit
```

## 배포

GitHub Pages 배포 주소는 `https://kibak812.github.io/deep-signal/`입니다. 소스 기준 브랜치는 `main`이며, 공개 빌드는 GitHub Actions 워크플로 `.github/workflows/deploy-pages.yml`이 `npm test`, `npm run copy:audit`, `npm run build`, `npm run audit`를 통과한 뒤 `dist/`를 Pages artifact로 게시하는 방식으로 운영합니다. 빌드 스크립트는 Pages에서 정적 에셋을 그대로 서빙하도록 `dist/.nojekyll`을 함께 생성하고, macOS 메타데이터, 에셋 재생성용 `public/assets/generated-sources/` 원본 묶음, 런타임에서 쓰지 않는 컨셉 레퍼런스와 교체된 타이틀 SVG는 배포 산출물에서 제외합니다.

## 구현 범위

- 시작 화면, 새 런, 저장된 런 미리보기와 덮어쓰기 확인이 포함된 이어하기, 키보드 포커스와 `Esc` 취소를 지원하는 런 포기 확인, 설정, 게임 정보, 기록, 코덱스, 가이드 화면
- 시작 화면의 캐릭터 패널: `에코 다이버`의 주력 전략, 시작 체력, 에너지, 크레딧, 덱 수, 시작 유물, 시작 덱 구성 표시
- 게임 정보 화면의 조작 안내, 제작진, 크레딧, 이용 안내·라이선스
- 위험과 보상, 다음 분기, 적 후보, 보스 특징, 상점과 휴식에서 얻을 수 있는 자원을 설명하는 절차 생성 경로 맵. 각 막은 초반 성장, 후반 엘리트·상점 선택, 휴식, 보스전으로 이어집니다.
- 카드 71장, 보상 카드 63장, 유물 31종, 일반 적 15종, 엘리트 5종, 보스 3종, 이벤트 20종
- 뽑기/손패/버림/소멸 더미, 전투 중 더미 점검 모달, 에너지, 방어도, 상태 이상, 적 의도, 손패 카드별 사용 후 변화 미리보기, 이번 턴 추천 카드와 추천 이유 표시, 6장 이상 손패도 데스크톱에서 잘리지 않는 부채꼴 배치
- 카드 보상 선택 전 추천 카드/추천 유물/고른 뒤 덱 변화를 상단에 짧게 보여주는 보상 요약, 덱 크기/평균 비용/중복/종류 비율 변화 안내, 카드별 받기/스킵 비교와 받지 않아도 좋은 상황 안내, 추천 보상 표시, 엘리트/보스/특수 전투 후 유물 3개 중 1개 선택과 추천 유물 및 현재 덱에 맞는 이유 안내, 카드 구매, 제거, 강화, 회복, 유물 구매, 상점 구매 뒤 잔액/덱 맞춤도/정비 결과 안내, 상점과 휴식 버튼의 추천 표시, 이벤트 선택지의 위험/보상과 추천 선택 표시, 이벤트/마켓/세이프룸의 현재 위치와 추천 판단 요약, 보스 직전 강화/제거 대상 선택에서 연속 방어·마무리·정화·카드 뽑기 부족을 함께 보여주는 카드 선택 브리핑과 안전한 취소 초기 포커스, 선택 직후 체력·크레딧·덱·유물 변화 토스트
- 6단계 난이도, 난이도별 클리어/최고층/최근 시드 진행 기록, 커스텀 시드, 날짜/난이도별로 바뀌는 실제 규칙 조항 2개가 적용되는 일일 계약 런, 일일 계약 전용 승률/최고층/최근 조항 기록, 기록/요약 기반 시드 재도전, 기록 화면의 다음 목표 제안, 승리 후 다음 난이도 바로 도전, 구역별 적 풀, 난이도별 보스 후 구역 전환 회복, 자동 저장 상태 표시와 로컬 저장/이어하기 미리보기, 자동 저장 백업 복구와 손상 저장 알림, 저장된 런 보호를 위한 새 런 덮어쓰기 확인과 저장 삭제 확인, 중도 런 포기 확인과 포기 요약 기록, 원인 진단과 다음 런의 3가지 짧은 행동을 먼저 보여주는 런 요약, 전적 기록

- 게임 가이드: 첫 런에서 자주 막히는 경로 선택, 전투, 덱 정비, 보스 대비 판단과 여섯 가지 주력 전략을 시작 화면과 런 도중 언제든 확인
- 현재 덱 화면의 핵심 카드 분석, 평균 비용/강화/저주/덱 크기, 카드 종류 분포, 핵심 카드와 다음 보상에서 찾을 역할, 덱 정비 힌트
- 효과음과 배경음의 독립 볼륨, 사운드 미리듣기, 효과음 중 음악을 짧게 낮췄다가 복귀시키는 믹싱, 애니메이션 속도, 텍스트 크기, 고대비, 플레이 힌트 표시 설정
- 전투 중 선택 대상 표시, 가장 먼저 처리할 위험을 크게 보여주는 적 의도 패널, 좌측 하단의 에너지/더미 HUD, 카드별 피해/방어/뽑기/상태/자원 변화 미리보기, 그림 위 종류 배지와 첫 효과 강조, 숫자키 카드 사용/방향키 대상 전환/키보드 턴 종료, 작은 화면용 적/손패 터치 배치
- 층/전투 타입에 따라 바뀌는 전투 배경 아틀라스, 이벤트 전용 장소 아틀라스, 카드 내부 일러스트 전용 아틀라스, 적/보스 스프라이트 아틀라스, 드래그 시 카드에서 대상까지 이어지는 조준선, 카드 사용과 적 행동을 대상 방향에 맞춘 얇은 궤적·피격 섬광·숫자 피드백으로 보여주는 전투 FX, 처치 확인 뒤 보상으로 넘어가는 승리 연출, 카드/방어/피해/유물 발동 배너, 유물 상세/발동 시점/현재 덱에 맞는 이유를 보여주는 모달, 보상, 상점, 이벤트, 승리/패배용 브라우저 내장 효과음과 메뉴/탐사/전투/보스/승패용 음악 반복

## 조작

- 카드 클릭: 선택한 적에게 사용
- 카드 드래그: 마우스, 터치, 펜으로 원하는 적 위에 놓아 대상 지정과 사용을 동시에 수행
- `1-9`, `0`: 손패의 해당 위치 카드 사용
- 적 클릭: 대상 지정
- `←` / `→`: 전투 중 대상 전환
- `E` 또는 `Space`: 전투 중 턴 종료
- 전투 더미 카운트 클릭: 뽑기/손패/버림/소멸 더미의 카드, 평균 비용, 강화 수, 다음에 뽑을 카드 확인
- `턴 종료`: 적 행동 후 다음 턴 시작
- 맵 노드 클릭: 다음 장소 이동
- 시드 입력: 같은 시드로 동일한 맵 구조의 런 재도전
- `오늘의 계약`: 현재 날짜와 난이도에 고정된 시드와 계약 조항으로 일일 챌린지 시작
- 난이도 선택: 시작 화면의 캐릭터 패널에서 적용 후 시작 체력을 즉시 확인
- `D`: 런 도중 현재 덱 열기/닫기
- `Esc`: 덱/선택 모달과 위험 행동 확인 닫기 또는 설정/코덱스/기록에서 이전 화면으로 복귀
- 손패 카드 미리보기: 카드 하단과 툴팁에서 사용 후 피해, 방어, 뽑기, 상태, 자원 변화 확인
- 카드/상태 툴팁: 카드와 전투 상태 위에 마우스를 올려 키워드와 상세 효과 확인
- 유물 줄 클릭: 현재 보유 유물의 희귀도, 발동 시점, 효과, 현재 덱에 맞는 이유 확인
- `코덱스`: 런 중에도 카드, 유물, 적 행동, 키워드와 상태 효과 확인
- `가이드`: 런 중에도 초반 보상에서 주력 정하기, 생존 판단, 덱 정비, 보스 대비 요령, 주력 전략 빠른 보기 확인

## 세계관

플레이어는 `에코 다이버`가 되어 가라앉은 데이터 해역의 최심부로 내려갑니다. 카드는 기억 조각, 타이드 신호, 감염된 인덱스, 심해 장치로 구성되며, 유물은 탐사의 규칙을 뒤트는 장비입니다. 세 구역의 보스는 각기 다른 방식으로 현재 덱의 강점과 빈틈을 드러내며, 최종 심층에서는 `마지막 문 성가대`가 생존력과 마무리 수단을 함께 확인합니다.

## 주요 시스템

- 전투: 에너지, 뽑기/손패/버림/소멸 더미, 방어도, 취약, 약화, 균열, 바이러스, 표식, 전하, 집중, 반격, 도금, 보존, 소멸, 생성, 비용 변화가 실제 규칙으로 동작합니다. 전투 화면은 상단에 현재 위치와 적 의도, 중앙에 가장 먼저 처리할 위험, 좌측 하단에 에너지·더미 HUD, 하단에 손패와 대상 안내를 배치해 “무엇이 위험한가”와 “지금 어떤 카드를 쓸 수 있는가”가 같은 흐름으로 읽히게 합니다. 에너지가 부족한 카드는 비용 표시부터 잠겨 보이도록 처리해 카드 사용 가능 여부를 바로 알 수 있습니다. 턴 종료 버튼은 기본 상태에서 핵심 결과만 짧게 보여주고, 마우스를 올리거나 키보드 포커스를 맞추면 이번 상대 턴에 체력이 얼마나 줄어드는지와 방어 여부를 작은 설명으로 보강합니다. 턴 종료를 누르면 적 턴 배너와 함께 입력이 잠기고, 짧은 간격 뒤 실제 적 행동과 피해·방어·상태 결과가 해결된 다음 내 턴 배너로 손패가 다시 활성화됩니다. 턴 요약 패널은 현재 손패와 에너지를 기준으로 체력 손실을 막을 수 있는지, 어느 적을 처치할 수 있는지, 카드 뽑기와 에너지 여유가 어떤지 짧게 요약하고, 세부 판단을 열면 추천 사용 순서와 각 카드의 이유·효과를 함께 보여줍니다. 더미 점검 모달은 카드 목록, 평균 비용, 강화 수, 주요 종류, 다음에 뽑을 카드를 보여줘 핵심 카드를 다시 찾을 시점을 판단하게 합니다.
- 적 의도: 공격, 방어, 강화, 해로운 상태 부여, 충전 예고, 소환, 보스 전용 행동, 단계 전환 체력과 현재 단계가 의도 표시와 전투 로그에 나타납니다.
- 덱 빌딩: 시작 화면에서 캐릭터의 시작 덱 구성과 주력 전략을 확인한 뒤 런에 들어갑니다. 시작 덱은 공격/방어/전하/표식/정화 카드를 모두 포함합니다. 런 중 현재 덱 화면은 어떤 카드로 이길지, 평균 비용, 강화/저주 수, 덱 크기, 카드 종류 분포, 정비 힌트를 표시합니다. 전투 보상 3장 선택/받지 않기, 엘리트/보스/특수 전투 후 유물 3개 중 1개 선택, 상점 구매, 카드 제거, 휴식 강화, 이벤트 변환/복제/제거가 연결되어 있습니다. 보상 화면은 현재 덱과 유물에 맞는 이유, 선택 후 덱 크기와 평균 비용 변화, 중복 부담, 종류 비율, 이번 보상을 받지 않아도 좋은지, 유물별 현재 덱에 맞는 이유와 발동 시점을 함께 보여줍니다. 이벤트, 마켓, 세이프룸은 현재 막과 보스까지 남은 거리, 이번 선택의 추천 이유를 짧은 카드로 먼저 보여줘 선택지가 메뉴처럼 흩어지지 않게 합니다. 선택을 확정하면 체력, 크레딧, 덱 장수, 강화, 유물 획득 변화가 짧은 결과 토스트로 떠서 방금 결정한 의미를 바로 확인할 수 있습니다.

## 출시 전 우선순위

- 장시간 플레이를 통한 각 난이도별 승률 조정. 이번 조정으로 압력층의 적 체력 배율을 1.17에서 1.18로 올리고 피해 배율은 1.08로 유지해, 피해 라운딩을 과하게 흔들지 않으면서 중간 난이도 압력을 보강했습니다. 최심층은 적 체력 배율을 1.32에서 1.30으로, 피해 배율을 1.20에서 1.18로 낮춘 뒤, 시작 최대 체력 페널티를 -14에서 -12로, 보상 크레딧 배율을 0.80에서 0.84로, 보스 후 회복 배율을 0.35에서 0.38로 조정해 중반 보스 이후 상층으로 올라갈 기회를 늘렸습니다. 2막 보스 익사한 알고리즘은 체력을 166-184에서 160-178로 낮춰 패턴 긴장감은 유지하되 최심층 중반 체력 벽을 줄였습니다. 대신 최심층의 마지막 문 성가대에는 최종 보스 전용 체력 배율 1.21을 적용해 중반 통과율을 올리면서도 최종 관문은 어렵게 남겼습니다. 자동 조종이 보스 본체를 `templateId` 문자열이 아니라 실제 적 티어로 판정하게 고쳐, 마지막 문 성가대 전투에서 소환수보다 본체 마무리를 우선합니다. 문지기 호출로 나오는 거울 해파리 체력 배율도 0.7에서 0.6으로 낮춰 2단계 소환이 긴장감은 주되 보스 본체를 과하게 가리지 않게 했습니다. 이번 갱신으로 18/36시드 리포트가 난이도별 최종 보스 도달, 보스전 승률, 진입 체력 비율, 패배 덱의 연속 방어 평균, 레퀴엠 턴 손패 방어 공백을 함께 기록합니다. 보스 상태 스트립은 마지막 문 성가대 2페이즈에서 문 낙하→호출→레퀴엠 순서를 직접 보여주고 현재 행동에 맞는 방어/체력 보존 문구를 띄웁니다. 보스 접근 브리핑은 마지막 문 앞에서 `큰 방어`와 `연속 방어`를 분리해, 문 낙하를 막는 단타 방어와 레퀴엠까지 이어 받을 보존/도금/약화 수단이 따로 부족한지 보여줍니다. 보상 추천은 두 지표가 동시에 낮을 때 `보스 대비 방어 묶음` 카드를 더 강하게 추천해, 마지막 문 앞 선택지가 실제 패턴 대비와 바로 연결되게 했습니다. 전투 손패 위에는 레퀴엠 본체 예상 피해, 보존 방어 수, 도금, 이번 턴 손패 방어를 압축한 대비 패널을 추가해 레퀴엠 손패 공백을 플레이 중 바로 읽게 했습니다. 36시드 장시간 리포트에서는 216런 모두 진행 불가 없이 완료됐고, 최종 보스 도달 185런 중 142승, 자동 조종 전체 승률 65.7%를 기록했습니다. 최심층은 14/36승, 최종 보스 도달 26런 중 14승이며, 중반 강적 사망은 9건에서 6건으로 줄었습니다. 이번 갱신으로 1단계 마지막 문 성가대 전투에서 손패가 2단계 전환선까지만 밀 수 있고 본체 처치는 못 하는 상황을 감지해, 2단계 진입 전 마무리 카드 보존 안내를 전투 힌트와 손패 패널에 함께 표시합니다. `npm run audit`는 이제 최종 보스 추천/경고 피로도 게이트를 별도 검사해 마지막 문 앞 방어 추천 비중, 마무리 부족 시 방어 추천 0건 유지, 방어가 부족하지 않은 상황의 방어 추천 비중, 보존 경고의 평균/최대 빈도가 허용 범위를 벗어나면 실패합니다. 다음 조정은 실제 플레이와 추가 자동 표본에서 이 게이트가 놓치는 체감 피로도와 최종 보스 앞 보상 다양성을 더 확인하는 일입니다.
- 카드별/적별 아트의 수작업 일러스트 다듬기와 추가 애니메이션. 이번 갱신으로 전투 스프라이트가 카드 사용, 적 공격, 피격, 처치 순간에 짧은 동작 잔상과 발밑 충격을 내도록 추가했고, 카드 일러스트에는 카드별 전경 모티프를 더해 같은 계열 카드도 한눈에 구분되게 했습니다. 보스는 2단계 진입 시 전용 PNG 스프라이트로 바뀌도록 연결했고, 일반 적과 엘리트는 전투판에서 더 잘 읽히도록 적별 크기·포즈 보정을 적용했습니다. 3체 이상 조우에는 다수 조우 요약 스트립을 추가해 총 예고 피해, 특수 행동 수, 현재 대상을 한 줄로 압축했고, 4체 검증 시나리오의 카드/의도판 겹침을 브라우저 검증으로 고정했습니다. 이번 갱신으로 적 스프라이트에 실루엣 후광과 림 레이어를 얹고, 전투 FX 중 비참여 적을 살짝 감쇠해 공격 주체와 피격 대상이 더 선명하게 남도록 했습니다. 다음 단계는 실제 플레이 중 이펙트 밝기와 적별 실루엣을 장시간으로 더 다듬는 일입니다.
- 음악 믹싱과 장기 플레이 청감 검증. 이번 갱신으로 테마 전환 프레이즈, 긴 브리지 프레이즈, 기본 필터/컴프레서 처리에 더해 효과음 강도 기반 음악 덕킹과 `audio-mix-report` 검증을 추가했습니다. 다음 단계는 실제 스피커와 헤드폰에서 보스전, 상점, 승패 화면을 오래 들어보며 테마별 상대 볼륨을 더 다듬는 일입니다.
- 전체 런 반복 검증, 모바일/태블릿 세부 터치 조작 다듬기. 이번 갱신으로 작은 화면 전투 보조 패널에 이전/다음 터치 대상 전환 버튼을 추가했고, 모바일/태블릿 브라우저 검증이 버튼 크기, 겹침, 실제 선택 변경을 기록합니다. 다음 단계는 실제 기기에서 긴 런 중 손패 스크롤과 대상 전환을 함께 반복 조작해 보는 일입니다.
